# 国际化 (i18n) 设计文档

## 概述

为 Obsidian Halo Plus 插件添加中英文双语支持，跟随 Obsidian 语言设置自动切换。

## 需求

1. 支持中文和英文两种语言
2. 跟随 Obsidian 语言设置自动切换
3. 国际化所有 UI 文本（设置页面、模态框、状态栏、通知、命令名称、右键菜单等）
4. 使用 JSON 文件管理翻译字符串
5. 轻量级实现，不引入外部依赖

#### 2. i18n 管理器 (index.ts)

```typescript
import en from './en.json';
import zh from './zh.json';

type Locale = 'en' | 'zh';
type TranslationKeys = typeof en;

class I18nManager {
  private locale: Locale = 'en';
  private translations: Record<Locale, TranslationKeys> = { en, zh };

  init(app: App): void {
    // 从 Obsidian 配置获取语言设置
    const obsidianLocale = (app as any).vault?.config?.locale || 'en';
    this.locale = obsidianLocale.startsWith('zh') ? 'zh' : 'en';
  }

  t(key: string, params?: Record<string, string>): string {
    const keys = key.split('.');
    let value: any = this.translations[this.locale];
    
    for (const k of keys) {
      value = value?.[k];
    }
    
    if (typeof value !== 'string') {
      return key; // 回退到键名
    }
    
    // 替换参数
    if (params) {
      return value.replace(/\{(\w+)\}/g, (_, key) => params[key] || '');
    }
    
    return value;
  }

  getLocale(): Locale {
    return this.locale;
  }
}

export const i18n = new I18nManager();
export const t = (key: string, params?: Record<string, string>) => i18n.t(key, params);
```

## 关键实现细节

### 语言检测

Obsidian 的语言设置存储在 `app.vault.config` 中，但该属性可能不存在或类型不明确。使用以下方式检测：

```typescript
// 在插件 onload 中
init(app: App): void {
  // 尝试从 Obsidian 配置获取语言设置
  // Obsidian 的语言设置可能在不同位置，需要兼容处理
  let obsidianLocale = 'en';
  
  // 方法1: 通过 vault.config (Obsidian 1.4+)
  if (app.vault && 'config' in app.vault) {
    const config = (app.vault as any).config;
    if (config && typeof config.locale === 'string') {
      obsidianLocale = config.locale;
    }
  }
  
  // 方法2: 通过 localStorage 备用
  if (obsidianLocale === 'en') {
    try {
      const savedLocale = localStorage.getItem('obsidian-locale');
      if (savedLocale) {
        obsidianLocale = savedLocale;
      }
    } catch (e) {
      // localStorage 可能不可用
    }
  }
  
  // 方法3: 通过 navigator.language 最终备用
  if (obsidianLocale === 'en') {
    obsidianLocale = navigator.language || 'en';
  }
  
  this.locale = obsidianLocale.startsWith('zh') ? 'zh' : 'en';
}
```

### 参数替换

支持带参数的翻译，例如：
```json
{
  "published": "Published: {title}"
}
```

使用方式：
```typescript
t('notices.published', { title: 'My Post' })
// 输出: "Published: My Post" 或 "已发布: My Post"
```

### 回退机制

如果翻译键不存在，回退到键名本身，确保不会出现 undefined。

## 注意事项

1. **不引入外部依赖** - 使用简单的 JSON 文件和自定义函数
2. **类型安全** - 使用 TypeScript 确保翻译键的正确性
3. **性能优化** - 翻译文件在插件加载时一次性加载
4. **维护性** - 翻译文件结构清晰，便于后续添加新语言
5. **Obsidian 兼容性** - 使用 Obsidian 的 API 获取语言设置

## 验收标准

1. 所有 UI 文本都显示为对应语言
2. 语言切换自动跟随 Obsidian 设置
3. 无翻译键时回退到键名
4. 支持参数替换
5. 不影响现有功能
